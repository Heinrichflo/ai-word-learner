# 开发工作流 - 现代化二次开发

## 新方针

找 GitHub 上高星标但长期无人更新的项目，进行现代化二次创作。

## 具体做法

1. **找项目**：star多但1年+没更新
2. **现代化**：用新语言重写
3. **增功能**：补全缺失功能
4. **精简代码**：重构优化

## 执行流程

1. research 每日调研合适项目
2. dev 评估可行性
3. content 写宣传文案
4. ops 推广

## 示例

- Python项目 → TypeScript/Go重写
- 老旧UI → 现代React/Vue
- 缺API → 加上
- 无测试 → 补测试
